Typical pipeline command:

  nextflow run genomic-medicine-sweden/metaval -profile <docker/singularity/.../institute> --input samplesheet.csv --outdir <OUTDIR>

help message of that parameter will be printed.  
or `--helpFull`.  

Input/output options
  --input                       [string] Path to comma-separated file containing information about the samples in the experiment. 
  --outdir                      [string] The output directory where the results will be saved. You have to use absolute paths to storage on 
Cloud infrastructure.  
  --email                       [string] Email address for completion summary. 
  --multiqc_title               [string] MultiQC report title. Printed as page header, used for filename if not otherwise specified. 

Verify identified species options
  --perform_verify_species      [boolean]         Extract reads assigned to taxonomic IDs. 
  --taxid                       [string]          A file containing a list taxids and corresponding species name separated by tabs. 
  --extract_kraken2_reads       [boolean]         Extract reads classified by kraken2 according to taxonomic ID. 
  --extract_centrifuge_reads    [boolean]         Extract reads classified by centrifuge according to taxonomic ID. 
  --extract_diamond_reads       [boolean]         Extract reads classified by DIAMOND according to taxonomic ID. 
  --phages_taxid                [string]          Path to a file containing a list of taxid taht should be excluded from downstream analysis 
  --evalue_threshold            [number, integer] An expected value to report an alignment during BLAST 
  --taxid2genome                [string]          Path to a file containing taxonomic IDs, organism name and corresponding genome paths. 
  --perform_shortread_denovo    [boolean]         Turn on de novo assembly for short reads 
  --perform_longread_denovo     [boolean]         Turn on de novo assembly for long reads 
  --flye_mode                   [string]          It provides `Flye` assembly modes. Our ONT reads were pre-processed by Guppy, so `--nano-corr` is 
the appropriate mode to use, as it is designed for ONT reads that have been corrected with other 
methods (with <3% error) [default: --nano-corr]  

Pathogen screening options
  --perform_screen_pathogens    [boolean] Screen viral pathogens for each sample 
  --pathogens_genomes           [string]  Path to a concatenated genome file containing complete genomes from a list of viral pathogens. 
 
  --accession2taxid             [string]  Path to a file containing the accession IDs, corresponding taxIDs and organism names for each 
pathogen genome.  
  --perform_longread_consensus  [boolean] Call consensus for long reads mapped to a pathogen genome. 
  --longread_consensus_tool     [string]  Specify the tool used for consensus calling of long reads.  (accepted: medaka, samtools) 
[default: medaka]  
  --perform_shortread_consensus [boolean] Call consensus for short reads mapped to a pathogen genome. 
  --consensus_min_bases         [number]  The minimum number of bases used to filter consensus sequences. The default is 50. [default: 50] 
 

General options
  --skip_blastn                 [boolean] Skip BLASTN of extracted reads. 
  --blastn_db                   [string]  Path to directory or tar.gz archive for pre-built BLAST database. 
  --blast_header                [string]  A header file for BLASTN/BLASTX hits. 
  --skip_blastx                 [boolean] Skip DIAMOND/BLASTX of extracted reads. 
  --blastx_db                   [string]  Path to directory or tar.gz archive for pre-built DIAMOND/BLASTX database. 
  --blastn_min_qlen             [integer] The minimum query length used to filter the BLASTN results [default: 50] 
  --blastx_min_qlen             [integer] The minimum query length (--min_qlen) used to filter the DIAMOND/BLASTX results [default: 50] 
 
  --blastn_min_pident           [number]  The minimum percentage of identical matches (--min_pident) used to filter BLASTN results 
[default: 50]  
  --blastx_min_pident           [number]  The minimum percentage of identical matches (--min_pident) used to filter DIAMOND/BLASTX results 
[default: 50]  
  --blastn_min_length           [integer] The minimum alignment length (--min_length) used to filter BLASTN results [default: 50] 
  --blastx_min_length           [integer] The minimum alignment length (--min_length) used to filter DIAMOND/BLASTX results [default: 50] 
 
  --blastn_max_evalue           [number]  The maximum e-value (--max_evalue) used to filter BLASTN results [default: 0.001] 
  --blastx_max_evalue           [number]  The maximum e-value (--max_evalue) used to filter DIAMOND/BLASTX results [default: 0.001] 
  --perform_mapping             [boolean] Turn on mapping 
  --min_read_counts             [integer] Set the read count threshold for performing de novo assembly or consensus calling. 

Generic options
  --multiqc_methods_description [string]          Custom MultiQC yaml file containing HTML including a methods description. 
  --help                        [boolean, string] Display the help message. 
  --help_full                   [boolean]         Display the full detailed help message. 
  --show_hidden                 [boolean]         Display hidden parameters in the help message (only works when --help or --help_full are provided).
